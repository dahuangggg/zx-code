## 一、问题引入

learn-claude-code s02 教了工具系统的三个核心：
1. **Tool Schema** — 告诉模型"你能调什么"
2. **Dispatch Map** — 收到调用请求后路由到正确的处理函数
3. **Path Sandboxing** — 防止工具逃逸到不该访问的地方

```python
# learn-claude-code 的极简 dispatch
TOOL_HANDLERS = {
    "bash":       run_bash,
    "read_file":  read_file,
    "write_file": write_file,
    "edit_file":  edit_file,
}

def execute_tool(name, arguments):
    handler = TOOL_HANDLERS[name]
    return handler(**arguments)
```

生产环境的工具系统要复杂得多——要处理工具数量爆炸、动态工具注册、并发安全、输入验证、输出截断等问题。

### 3.2 Codex — Rust Struct + JSON Schema

```rust
// codex-rs/tools/src/tool_registry.rs
pub struct ToolSpec {
    pub name: String,
    pub description: String,
    pub parameters: serde_json::Value,  // JSON Schema
    pub strict: bool,                   // 严格模式验证
}

// 工具注册表——动态收集所有工具
pub struct ToolRegistry {
    tools: HashMap<String, Box<dyn ToolHandler>>,
    mcp_tools: Vec<McpToolSpec>,      // 从 MCP 服务器发现的
    skill_tools: Vec<SkillToolSpec>,  // 从技能目录加载的
}

impl ToolRegistry {
    pub fn resolve(&self, name: &str) -> Option<&dyn ToolHandler> {
        // 先查内置，再查 MCP，最后查 Skills
        self.tools.get(name)
            .or_else(|| self.mcp_tools.find(name))
            .or_else(|| self.skill_tools.find(name))
    }
}
```

**Codex 的关键：分层查找** — 内置 > MCP > Skills，保证核心工具不被覆盖。

### 3.3 Pi-Mono — TypeBox Schema + Execute 函数

```typescript
// packages/agent/src/types.ts
interface AgentTool {
    name: string;
    label: string;                      // 人类可读名称
    description: string;
    parameters: TObject;                // TypeBox schema（编译时类型安全）
    execute: (args: any) => Promise<ToolResult>;
    prepareArguments?: (args: any) => any;  // 预处理参数
}

// packages/coding-agent/src/core/tools/read.ts — 示例
const readTool: AgentTool = {
    name: "read",
    label: "Read File",
    description: "Read file contents with optional line range",
    parameters: Type.Object({
        file_path: Type.String(),
        offset: Type.Optional(Type.Number()),
        limit: Type.Optional(Type.Number()),
    }),
    async execute({ file_path, offset, limit }) {
        const content = await fs.readFile(file_path, "utf-8");
        // 截断逻辑...
        return { content: truncated, details: { lines, truncated } };
    }
};
```

**Pi-Mono 的关键：TypeBox** — 用 TypeScript 类型系统生成 JSON Schema，编译时就能检查参数是否正确。

### 3.4 Claude Code — Zod Schema + 丰富接口

```typescript
// src/Tool.ts
interface Tool {
    name: string;
    description(input: any, context: ToolContext): string;  // 动态描述！
    prompt(): string;                // 写入 system prompt 的使用说明
    inputSchema: z.ZodSchema;        // Zod 严格验证
    outputSchema: z.ZodSchema;
    
    // 并发安全性判断
    isConcurrencySafe(input: any): boolean;
    
    // 权限检查（在执行前）
    checkPermissions(input: any, context: ToolContext): PermissionDecision;
    
    // 执行
    execute(input: any, context: ToolContext): Promise<ToolOutput>;
    
    // 可选的生命周期钩子
    onBeforeExecute?(input: any): void;
    onAfterExecute?(output: ToolOutput): void;
}
```

**Claude Code 的关键**：
- **动态描述** — `description()` 是函数不是字符串，可以根据输入生成不同的描述
- **`prompt()`** — 每个工具自带使用说明，会注入到 system prompt
- **`isConcurrencySafe()`** — 每个工具自己声明自己是否可以并行
- **`checkPermissions()`** — 权限检查内置在工具接口里

## 五、输入验证与输出截断

### 5.1 输入验证

```
learn-claude-code: 无验证（信任模型）
Codex:             JSON Schema strict 模式（模型必须精确匹配 schema）
Pi-Mono:           TypeBox → JSON Schema 验证（validateToolArguments）
Claude Code:       Zod schema（inputSchema.parse(input)，不匹配直接报错）
```

### 5.2 输出截断

这是生产系统的重要问题——如果 `cat` 了一个 10MB 的文件怎么办？

```typescript
// Codex — 硬限制 + 摘要
const MAX_OUTPUT = 100_000; // 字符
if (output.length > MAX_OUTPUT) {
    return truncate(output, MAX_OUTPUT) + "\n[output truncated]";
}

// Pi-Mono — TruncationResult 统一类型
interface TruncationResult {
    content: string;
    truncated: boolean;
    totalLines: number;
    shownLines: number;
}
// 每个工具自己决定截断策略

// Claude Code — 多层截断
// 1. 工具层：每个工具自己截断（如 read_file 默认 2000 行）
// 2. 执行层：StreamingToolExecutor 检查总输出大小
// 3. 系统层：超大结果写到磁盘，context 中只放摘要
```

## 六、路径安全（Path Sandboxing）

learn-claude-code 的 `safe_path()` 只是基础：

```python
def safe_path(p: str) -> str:
    resolved = os.path.abspath(p)
    if not resolved.startswith(ALLOWED_DIR):
        raise ValueError(f"Access denied: {resolved}")
    return resolved
```

三个仓库的实现要复杂得多：

```
┌─────────────────────────────────────────────────────────┐
│ Codex: OS 级沙箱 + 文件系统策略                          │
│                                                         │
│ FileSystemSandboxPolicy {                               │
│   readable_roots: ["/", "~"]          # 可读的目录      │
│   writable_roots: ["./", "/tmp"]      # 可写的目录      │
│   denied_paths: [".git", ".codex"]    # 禁止访问的      │
│ }                                                       │
│                                                         │
│ 执行方式:                                                │
│   macOS: sandbox-exec -p '(deny default)(allow ...)'    │
│   Linux: bwrap --ro-bind / / --bind ./project ./project │
│   Windows: CreateRestrictedToken(...)                    │
│                                                         │
│ → 操作系统强制执行，即使工具代码有 bug 也逃不出去        │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ Pi-Mono: 无内置沙箱                                      │
│                                                         │
│ 设计哲学: 用户自己负责                                    │
│   - 在容器里运行                                         │
│   - 通过 SSH Extension 远程执行                          │
│   - BashOperations 接口可以被替换                        │
│                                                         │
│ interface BashOperations {                               │
│   execute(cmd, opts): Promise<Result>                   │
│   // 默认实现 = Node.js spawn                            │
│   // SSH 扩展替换为远程执行                              │
│ }                                                       │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ Claude Code: 权限规则 + 钩子                             │
│                                                         │
│ 不用 OS 沙箱，而是靠多层权限规则:                        │
│ 1. 路径模式匹配 (glob)                                  │
│ 2. 工具名匹配                                           │
│ 3. 命令模式匹配                                         │
│ 4. 用户交互确认                                         │
│                                                         │
│ 加上 hooks 系统可以在工具执行前做任意检查                │
└─────────────────────────────────────────────────────────┘
```

## 七、工具热注册（动态工具）

learn-claude-code 的 dispatch map 是启动时固定的。生产系统需要运行时动态添加工具：

```typescript
// Codex — MCP 工具自动发现
// MCP 服务器启动时，Codex 枚举它的工具列表
const mcpTools = await mcpClient.listTools();
for (const tool of mcpTools) {
    registry.register(tool.name, mcpHandler(tool));
}
// 工具名变为: mcp__server_name__tool_name

// Pi-Mono — Extension API 注册
// 扩展可以在任何时候注册新工具
extensionContext.registerTool({
    name: "deploy",
    label: "Deploy to Production",
    description: "Deploy the current branch",
    parameters: Type.Object({ env: Type.String() }),
    async execute({ env }) {
        return await deploy(env);
    }
});

// Claude Code — Skills 按需加载
// 技能名字始终在 system prompt 里
// 但技能的完整内容只在模型调用 SkillTool 时才加载
// → learn-claude-code s05 的"两层加载"策略
```

## 八、关键差异总结

| 维度 | Codex | Pi-Mono | Claude Code |
|------|-------|---------|-------------|
| **工具定义** | Rust struct + JSON Schema | TypeBox schema | Zod schema |
| **路由层级** | 内置 > MCP > Skills | 扁平（name 唯一）| 内置 > MCP > Skills > Plugins |
| **验证** | strict JSON Schema | TypeBox 验证 | Zod parse |
| **沙箱** | OS 级（Seatbelt/Landlock）| 无（靠容器/扩展）| 权限规则层 |
| **并发** | Orchestrator 单个执行 | parallel/sequential 可选 | 安全分类后分批 |
| **截断** | 硬限制 | 工具自定义 | 多层（工具/执行器/系统）|
| **动态注册** | MCP + Skills | Extension API | MCP + Skills + Plugins |
| **审批集成** | 内置在 Orchestrator | beforeToolCall 钩子 | canUseTool() 7 层 |

## 九、给你的思考题

1. **为什么 Claude Code 要让每个工具自己实现 `isConcurrencySafe()`，而不是全局声明？**
   - 提示：想想 `bash` 工具——它的安全性取决于**具体命令**，不取决于工具名

2. **Pi-Mono 不做沙箱是"偷懒"还是"聪明"？**
   - 提示：想想 BashOperations 接口——只要替换这一个接口，就能实现任意执行后端

3. **Codex 的"沙箱升级重试"是什么意思？什么场景下有用？**
   - 提示：工具在严格沙箱下失败（比如需要网络），自动切换到宽松沙箱重试
