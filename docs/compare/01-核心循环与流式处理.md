# 01 — 核心循环与流式处理

> 对应 learn-claude-code: **s01 Agent Loop** + **s11 Error Recovery**

---

## 一、问题引入

learn-claude-code 用 30 行 Python 告诉你 Agent 的本质：

```python
# learn-claude-code s01 的极简版
while True:
    response = call_llm(messages)
    if no_tool_calls(response):
        break
    for tool_call in response.tool_calls:
        result = execute(tool_call)
        messages.append(tool_result(result))
```

这个 `while True` 就是一切的起点。但生产环境要回答更多问题：

1. **流式怎么处理？** — 用户不能等 30 秒才看到第一个字
2. **工具执行期间模型在干嘛？** — 阻塞还是继续？
3. **出错了怎么恢复？** — token 超限？网络断了？模型拒绝了？
4. **谁来驱动循环？** — 同步 while 还是事件驱动？

三个仓库给出了三种截然不同的答案。

---

## 二、核心概念回顾

learn-claude-code 定义了两个关键状态对象：

```
LoopState {
    messages: Message[]      # 对话历史
    turn_count: int          # 已执行轮数
    transition_reason: str   # 为什么进入下一轮
}

QueryState {
    tool_results_pending: bool  # 有工具结果等待回写
    should_compact: bool        # 需要压缩上下文
    should_continue: bool       # 是否继续循环
}
```

**核心洞察**：内容状态（messages）和流程控制状态（turn_count, should_compact）必须分离。

## 三、三个仓库的实现对比

### 3.1 Codex — Submission-Event 模型（事件驱动）

Codex 的循环**不是** while 循环，而是**事件驱动**的：

```
                 ┌──────────────────┐
  用户/SDK ──→   │  submit(Op)      │
                 └────────┬─────────┘
                          ↓
                 ┌──────────────────┐
                 │ submission_loop  │ ← 一个永驻的 tokio task
                 │ match sub.op {  │
                 │   UserTurn => …  │
                 │   Interrupt => … │
                 │   Shutdown => …  │
                 │ }               │
                 └────────┬─────────┘
                          ↓
                 ┌──────────────────┐
                 │ tx_event.send()  │ ──→ 客户端
                 └──────────────────┘
```

**伪代码**（简化自 `codex-rs/core/src/codex.rs`）：

```rust
// Codex 的核心循环
async fn submission_loop(rx_sub: Receiver<Submission>, tx_event: Sender<Event>) {
    loop {
        let sub = rx_sub.recv().await;    // 等待下一个操作
        match sub.op {
            Op::UserTurn { input, .. } => {
                // 进入 turn loop
                turn_loop(&input, &tx_event).await;
            }
            Op::ExecApproval { id, decision } => {
                // 用户批准了某个工具调用
                approval_channels[id].send(decision);
            }
            Op::Interrupt => {
                // 中断当前 turn
                cancel_token.cancel();
            }
            Op::Shutdown => break,
            // ... 22 种 Op
        }
    }
}

async fn turn_loop(input: &str, tx_event: &Sender<Event>) {
    loop {
        // 1. 构建上下文
        let context = build_context(&history, &memories);
        
        // 2. 流式调用模型（WebSocket 优先，SSE 降级）
        let mut stream = model_client.stream(context).await;
        
        // 3. 消费流事件
        while let Some(event) = stream.next().await {
            match event {
                StreamEvent::TextDelta(text) => {
                    tx_event.send(Event::Message(text));  // 实时推给客户端
                }
                StreamEvent::ToolCall(call) => {
                    // 4. 通过 Orchestrator 执行工具
                    let result = orchestrator.run(call).await;
                    // 结果写回历史
                    history.push(tool_result(result));
                }
                StreamEvent::Done => break,
            }
        }
        
        // 5. 如果没有更多工具调用，结束 turn
        if !has_pending_tools {
            tx_event.send(Event::TurnCompleted);
            break;
        }
        // 否则继续循环（带工具结果再调一次模型）
    }
}
```

**Codex 的关键设计**：
- **双通道流式**：优先 WebSocket（持久连接），失败降级到 HTTP/SSE
- **非阻塞**：所有 I/O 都是 tokio 异步，主循环永远不阻塞
- **Op 枚举统一入口**：22 种操作（用户输入、审批、中断、配置变更…）全部走同一个通道

### 3.2 Pi-Mono — Async Generator 模型（拉取式）

Pi-Mono 用 TypeScript 的 async generator 实现循环：

```
  Agent.run(prompt)
    ↓
  runAgentLoop*()        ← async generator，调用者 pull 事件
    ├── yield agent_start
    ├── loop {
    │     yield turn_start
    │     yield* streamAssistantResponse()  ← 流式 LLM 调用
    │     if (tool_calls) {
    │       yield* executeToolCalls()
    │     } else {
    │       yield turn_end
    │       break
    │     }
    │   }
    └── yield agent_end
```

**伪代码**（简化自 `packages/agent/src/agent-loop.ts`）：

```typescript
// Pi-Mono 的核心循环
async function* runAgentLoop(
    context: Context,
    state: AgentState
): AsyncGenerator<AgentEvent> {
    yield { type: "agent_start" };
    
    // 注入用户消息
    state.messages.push({ role: "user", content: prompt });
    
    while (true) {
        yield { type: "turn_start" };
        
        // 1. 流式调用 LLM（统一抽象，支持 15+ 供应商）
        const response = yield* streamAssistantResponse(context, state);
        
        // 2. 检查是否有工具调用
        const toolCalls = extractToolCalls(response);
        
        if (toolCalls.length === 0) {
            yield { type: "turn_end", reason: "stop" };
            break;
        }
        
        // 3. 执行工具
        yield* executeToolCalls(toolCalls, state);
        // executeToolCalls 内部会 yield tool_execution_start/end 事件
        // 工具结果自动追加到 state.messages
        
        yield { type: "turn_end", reason: "tool_use" };
        // 继续循环
    }
    
    yield { type: "agent_end" };
}

// 工具执行——支持并行和串行两种模式
async function* executeToolCalls(calls, state) {
    if (state.toolExecutionMode === "parallel") {
        // 先串行做 preflight（验证参数），再并发执行
        for (const call of calls) {
            yield { type: "tool_execution_start", call };
            // preflight 验证
        }
        const results = await Promise.all(
            calls.map(call => call.tool.execute(call.arguments))
        );
        for (const [i, result] of results.entries()) {
            yield { type: "tool_execution_end", call: calls[i], result };
            state.messages.push(toolResult(calls[i].id, result));
        }
    } else {
        // 串行执行
        for (const call of calls) {
            yield { type: "tool_execution_start", call };
            const result = await call.tool.execute(call.arguments);
            yield { type: "tool_execution_end", call, result };
            state.messages.push(toolResult(call.id, result));
        }
    }
}
```

**Pi-Mono 的关键设计**：
- **Generator 即协议**：调用者通过 `for await` 拉取事件，天然支持背压
- **Steering 消息队列**：可以在循环运行时注入额外消息

```typescript
// Pi-Mono 独有的 steering 机制
class Agent {
    private steeringQueue: Message[] = [];
    private followUpQueue: Message[] = [];
    
    // 外部可以在 agent 运行时注入消息
    steer(message: Message) {
        this.steeringQueue.push(message);
    }
    
    // 在每次 turn 开始时检查
    private drainSteering(): Message[] {
        const msgs = this.steeringQueue.splice(0);
        return msgs;  // 注入到下一次 LLM 调用的 messages 中
    }
}
```

### 3.3 Claude Code — Async Generator + React 模型

Claude Code 也用 async generator，但和 Pi-Mono 有本质区别——它是**在 React 组件里消费 generator**的：

```
  QueryEngine.submitMessage(prompt)
    ↓
  query*()                 ← async generator
    ├── yield SystemMessage
    ├── loop {
    │     stream Claude API
    │     parse tool_use blocks
    │     StreamingToolExecutor.run()
    │       ├── 并发安全分类
    │       ├── 权限检查
    │       └── 批量执行
    │     yield ToolResultMessage
    │   }
    └── yield AssistantMessage
    ↓
  React Ink 组件 for await (message of query()) {
      render(message)    ← 实时更新终端 UI
  }
```

**伪代码**（简化自 `src/query.ts`）：

```typescript
// Claude Code 的核心循环
async function* query(
    prompt: string,
    systemPrompt: string,
    tools: Tool[],
    state: AppState
): AsyncGenerator<Message | StreamEvent> {
    // 构建消息列表
    const messages = [...state.messages, userMessage(prompt)];
    
    while (true) {
        // 1. 调用 Claude API（流式）
        const stream = await claude.messages.stream({
            model: state.model,
            system: systemPrompt,
            messages: messages,
            tools: tools.map(t => t.toSchema()),
        });
        
        // 2. 收集助手回复，实时 yield 文本块
        const assistantBlocks = [];
        for await (const event of stream) {
            if (event.type === "content_block_delta") {
                yield { type: "stream", delta: event.delta };
            }
            if (event.type === "content_block_stop") {
                assistantBlocks.push(event.block);
            }
        }
        
        // 3. 提取工具调用
        const toolUses = assistantBlocks.filter(b => b.type === "tool_use");
        
        if (toolUses.length === 0) {
            yield assistantMessage(assistantBlocks);
            break;  // 没有工具调用，结束
        }
        
        // 4. 用 StreamingToolExecutor 批量执行
        const executor = new StreamingToolExecutor(toolUses, tools);
        const results = await executor.run({
            // 关键：按并发安全性分批
            // 读操作并行，写操作串行
            onPermissionNeeded: (tool, input) => canUseTool(tool, input),
            onProgress: (msg) => { yield msg; },
        });
        
        // 5. 工具结果写回 messages
        messages.push(assistantMessage(assistantBlocks));
        messages.push(toolResultMessage(results));
        
        // 继续循环
    }
}
```

**Claude Code 的关键设计**：
- **StreamingToolExecutor 并发安全分类**：

```typescript
// Claude Code 独有的并发安全分批执行
class StreamingToolExecutor {
    async run(toolCalls: ToolCall[]): Promise<ToolResult[]> {
        // 第一步：分类
        const safe = [];    // 可并行的（如 read_file, grep）
        const unsafe = [];  // 必须串行的（如 bash with &&, edit_file）
        
        for (const call of toolCalls) {
            if (this.tools[call.name].isConcurrencySafe(call.input)) {
                safe.push(call);
            } else {
                unsafe.push(call);
            }
        }
        
        // 第二步：并行执行安全的，串行执行不安全的
        const results = [];
        
        // 安全的批量并行（最多 10 并发）
        const safeResults = await Promise.all(
            safe.map(call => this.executeSingle(call))
        );
        results.push(...safeResults);
        
        // 不安全的逐个串行
        for (const call of unsafe) {
            results.push(await this.executeSingle(call));
        }
        
        return results;
    }
    
    private isConcurrencySafe(name: string, input: any): boolean {
        // bash 带管道/&&：不安全
        if (name === "bash" && /[|&;]/.test(input.command)) return false;
        // 读文件：安全
        if (name === "read_file") return true;
        // 编辑不同文件：安全
        if (name === "edit_file") return !this.hasConflict(input.path);
        return false;
    }
}
```

## 四、错误恢复对比（s11）

learn-claude-code 教了三种恢复分支：
1. **Continuation** — max_tokens 截断，注入提示继续
2. **Compaction** — 上下文溢出，压缩后重试
3. **Backoff** — 网络/API 错误，指数退避

三个仓库如何实现？

### 4.1 错误恢复伪代码对比

```
┌─────────────────────────────────────────────────────────────────┐
│                        Codex 的恢复策略                         │
├─────────────────────────────────────────────────────────────────┤
│ turn_loop:                                                      │
│   try:                                                          │
│     stream = model.stream(context)                              │
│     process(stream)                                             │
│   catch ContextOverflow:                                        │
│     compact(history)           # 调用远端 compact 服务或本地    │
│     retry()                    # 重新调用                       │
│   catch WebSocketError:                                         │
│     fallback to HTTP/SSE       # WebSocket → SSE 降级           │
│     retry()                                                     │
│   catch RateLimit:                                              │
│     sleep(backoff)             # 指数退避                       │
│     retry()                                                     │
│   catch ToolTimeout:                                            │
│     escalate_sandbox()         # 尝试更宽松的沙箱               │
│     retry()                                                     │
│                                                                 │
│ 特色: 传输层降级 (WS→SSE), 沙箱升级重试                        │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                       Pi-Mono 的恢复策略                        │
├─────────────────────────────────────────────────────────────────┤
│ runAgentLoop:                                                   │
│   try:                                                          │
│     response = yield* streamAssistantResponse()                 │
│   catch ContextOverflow:                                        │
│     yield { type: "compaction_start" }                          │
│     compactMessages(state.messages)                             │
│     yield { type: "compaction_end" }                            │
│     continue  // 重新进入循环                                   │
│   catch ProviderError:                                          │
│     yield { type: "auto_retry_start", attempt, delay }          │
│     await sleep(delay)                                          │
│     continue  // 自动重试                                       │
│   catch ToolError:                                              │
│     state.messages.push(errorResult(call.id, error))            │
│     continue  // 告诉模型工具出错了，让它自己处理               │
│                                                                 │
│ 特色: 事件驱动（yield 恢复事件），模型自愈（错误当工具结果）   │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                     Claude Code 的恢复策略                       │
├─────────────────────────────────────────────────────────────────┤
│ query loop:                                                     │
│   try:                                                          │
│     stream = claude.messages.stream(...)                        │
│   catch MaxTokens:                                              │
│     // 模型因 max_tokens 截断                                   │
│     messages.push(continuation_reminder)                        │
│     continue  // 让模型继续                                     │
│   catch ContextOverflow:                                        │
│     messages = await compact(messages)  # 内置压缩              │
│     continue                                                    │
│   catch OverloadError:                                          │
│     await exponentialBackoff(attempt)                            │
│     continue                                                    │
│   catch ToolError:                                              │
│     yield errorMessage(tool, error)                             │
│     // 工具结果包含错误，回写给模型                              │
│     messages.push(toolError(call.id, error.message))            │
│     continue                                                    │
│                                                                 │
│ 特色: continuation_reminder（提示模型未完成），token 预算跟踪    │
└─────────────────────────────────────────────────────────────────┘
```

## 五、关键差异总结

| 维度 | Codex | Pi-Mono | Claude Code |
|------|-------|---------|-------------|
| **循环模型** | 事件驱动 (Submission→Event) | Async Generator (pull) | Async Generator (pull) + React |
| **驱动方式** | `rx_sub.recv()` 被动等待 | `for await` 调用者拉取 | `for await` + React 渲染 |
| **流式传输** | WebSocket 优先 + SSE 降级 | 供应商 SDK 原生流 | Anthropic SDK 流 |
| **工具并发** | Orchestrator 单个执行 | 并行/串行可选 | 并发安全分类后分批 |
| **错误降级** | WS→SSE, 沙箱升级 | 事件通知 + 自动重试 | continuation + compact |
| **状态位置** | Rust struct (内存) | AgentState 对象 | Zustand store + React |
| **语言特性** | tokio async/await | TS async generator | TS async generator |
