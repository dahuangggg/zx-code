# 07 — 钩子、技能与扩展

> 对应 learn-claude-code: **s05 Skill Loading** + **s08 Hook System**

---

## 一、问题引入

一个 Agent 不可能把所有功能都内置。你需要让用户扩展它：

- **Hooks（钩子）**：在特定时机执行自定义代码（"工具执行前检查一下"）
- **Skills（技能）**：给模型注入领域知识（"你是 React 专家"）
- **Extensions（扩展）**：注册新工具、命令、UI（"添加部署功能"）
- **Plugins（插件）**：打包好的扩展集合（"安装这个 npm 包"）

learn-claude-code 把这些分成了两章：s05（技能加载）和 s08（钩子系统）。

## 二、s05 Skill Loading — 两层加载

### learn-claude-code 的教学

```python
# 关键洞察：技能名称便宜（放在 system prompt），技能内容贵（按需加载）

# 层 1：名称始终在 system prompt 里
SYSTEM_PROMPT += """
Available skills:
- agent-builder: Design and build agents
- code-review: Comprehensive code review  
- mcp-builder: Build MCP servers
"""
# → 模型知道有这些技能，但不知道详细内容

# 层 2：内容通过 tool_result 按需加载
def load_skill(name: str) -> str:
    """模型调用 Skill 工具时才加载完整内容"""
    content = read_file(f"skills/{name}/prompt.md")
    return content  # 作为 tool_result 返回给模型
    # → 只有在模型需要时才占用 context window
```

### 三个仓库的实现

### 2.1 Codex — Skills 目录

```rust
// codex-rs/skills/

// Codex 的 Skills 是目录结构:
// ~/.codex/skills/
// ├── deploy/
// │   ├── manifest.json    # 元数据
// │   └── prompt.md        # 技能内容
// └── review/
//     ├── manifest.json
//     └── prompt.md

// 加载方式：
// 1. 启动时扫描 skills 目录
// 2. 技能名称注入 system prompt
// 3. 模型调用时加载完整内容

// manifest.json 示例:
{
    "name": "deploy",
    "description": "Deploy to production environments",
    "auto_discover": true
}
```

### 2.2 Pi-Mono — Skills + Prompts + Pi Packages

```typescript
// Pi-Mono 有三种注入知识的方式：

// 1. Skills（和 learn-claude-code 类似）
// ~/.pi/agent/skills/ 或 .pi/skills/
// 目录中的 markdown 文件作为技能内容

// 2. Prompts（自定义提示词模板）

// 3. 已安装的 Pi Packages

// Pi-Mono 会自动读取项目根目录的 AGENTS.md
```

### 2.3 Claude Code — Skills + SkillTool

```typescript
// Claude Code 的 Skill 系统最完整：

// 1. 内置技能（随 Claude Code 分发）
// 例如: commit, review-pr, pdf 等

// 2. 自定义技能（用户创建）
// ~/.claude/skills/ 或 .claude/skills/

// 3. SkillTool — 在子代理中执行技能
class SkillTool implements Tool {
    name = "Skill";
    
    async execute({ skill, args }) {
        // 加载技能 prompt
        const skillContent = loadSkill(skill);
        
        // 在隔离的子代理中执行（重要！）
        // → 防止技能内容污染主对话上下文
        const result = await runInSubagent({
            systemPrompt: skillContent,
            userPrompt: args,
        });
        
        return result;
    }
}

// 关键设计：用户可以用 /skill-name 快捷调用
// 例如 /commit → 调用 commit 技能
// 例如 /review-pr 123 → 调用 review-pr 技能
```

## 三、s08 Hook System — 生命周期钩子

### learn-claude-code 的教学

```python
# 三个生命周期事件:
# SessionStart  — 会话开始时触发一次
# PreToolUse    — 每次工具执行前触发
# PostToolUse   — 每次工具执行后触发

# 退出码协议:
# exit 0 → 通过（可以附带注解）
# exit 1 → 阻止（PreToolUse 时阻止工具执行）
# exit 2 → 注解（PostToolUse 时附加信息）

# 示例: 禁止在生产分支上执行 git push
# hooks/pre_tool_use.sh:
# if [ "$TOOL_NAME" = "bash" ] && echo "$TOOL_INPUT" | grep -q "git push"; then
#     BRANCH=$(git branch --show-current)
#     if [ "$BRANCH" = "main" ]; then
#         echo "Blocked: cannot push to main directly"
#         exit 1
#     fi
# fi
# exit 0
```

### 三个仓库的 Hook 实现

### 3.1 Codex — Rust Hook 系统

```rust
// codex-rs/hooks/

// Codex 的 hooks 比较简单：
// 配置在 ~/.codex/config.toml 中
[hooks]
before_agent = "~/.codex/hooks/before.sh"
after_agent = "~/.codex/hooks/after.sh"

// 事件类型有限，主要是会话级别的
// 不像 learn-claude-code 那样有细粒度的 PreToolUse
// 因为 Codex 靠 ExecPolicy + Sandbox 来做工具级控制
```

### 3.2 Pi-Mono — Extension API（最灵活）

```typescript
// Pi-Mono 没有"钩子"概念，而是有更强大的 Extension API

// 扩展可以订阅的事件类型:
type ExtensionEvents = {
    // 工具生命周期
    tool_call: (call: ToolCall) => void;
    tool_result: (result: ToolResult) => void;
    
    // 消息生命周期
    message_update: (msg: AssistantMessage) => void;
    
    // 代理生命周期
    before_agent_start: () => void;
    before_provider_request: (req: Request) => void;
    
    // 用户交互
    user_bash: (cmd: string) => void;
    input: (text: string) => void;
    
    // 会话生命周期
    session_before_switch: () => void;
    session_before_fork: () => void;
    session_before_compact: () => void;
    session_before_tree: () => void;
    
    // 资源发现
    resources_discover: () => Resource[];
    context: () => ContextItems;
};

// 扩展注册示例:
export default function myExtension(ctx: ExtensionContext) {
    // 1. 注册工具
    ctx.registerTool({
        name: "deploy",
        async execute({ env }) { ... }
    });
    
    // 2. 注册命令
    ctx.registerCommand({
        name: "deploy",
        description: "Deploy to production",
        execute: async () => { ... }
    });
    
    // 3. 注册快捷键
    ctx.registerShortcut({
        key: "ctrl+d",
        action: () => ctx.runCommand("deploy"),
    });
    
    // 4. 订阅事件
    ctx.on("tool_call", async (call) => {
        if (call.name === "bash" && isDangerous(call.arguments.cmd)) {
            const ok = await ctx.confirm("确定要执行这个命令吗？");
            if (!ok) return { block: true };
        }
    });
    
    // 5. 自定义 UI
    ctx.setWidget("deploy-status", {
        position: "footer",
        render: () => `Deploy: ${status}`,
    });
}

// 加载方式:
// 1. ~/.pi/agent/extensions/my-ext.ts → 自动加载
// 2. .pi/extensions/project-ext.ts → 项目级
// 3. Pi Package 中的 extensions/ 目录
```

### 3.3 Claude Code — Hook + Plugin + Skill 三合一

```typescript
// Claude Code 的 Hook 系统

// Hook 类型:
type HookEvent = 
    | "session_start"
    | "pre_tool_use"
    | "post_tool_use"
    | "post_tool_use_failure"
    | "permission_request"
    | "stop"
    | "post_sampling";

// 配置方式 (~/.claude/settings.json):
{
    "hooks": {
        "pre_tool_use": [
            {
                "matcher": { "tool": "bash" },
                "command": "~/.claude/hooks/check-bash.js"
            }
        ],
        "post_tool_use": [
            {
                "matcher": { "tool": "*" },
                "command": "~/.claude/hooks/log-all.sh"
            }
        ]
    }
}

// Hook 脚本接收 JSON stdin，返回 JSON stdout:
// 输入: { tool: "bash", input: { command: "rm -rf /" } }
// 输出: { decision: "deny", reason: "Dangerous command" }

// 和 learn-claude-code 的区别:
// - learn-claude-code 用退出码 (0/1/2)
// - Claude Code 用 JSON 输入输出（更丰富）
// - Claude Code 有更多事件类型（7 种 vs 3 种）
// - Claude Code 支持 matcher 过滤（只对特定工具触发）
```

## 四、扩展机制全景对比

```
                    Codex          Pi-Mono         Claude Code
                    ─────          ───────         ───────────
注入知识            Skills         Skills          Skills
                    目录           + Prompts       + SkillTool
                                   + Pi Packages   (子代理执行)

拦截工具            ExecPolicy     Extension API   Hooks
                    (规则引擎)     (beforeToolCall) (pre_tool_use)

添加工具            MCP            Extension API   MCP + Plugins
                    (外部服务)     (registerTool)   (多种方式)

添加命令            无             Extension API   Plugins
                                   (registerCommand) + 内置命令

自定义 UI           TUI 配置       Extension API   React 组件
                                   (setWidget等)    (内置)

事件订阅            有限           全面            较全面
                    (2 个钩子)     (15+ 事件)      (7 种钩子)

包管理              无             Pi Packages     npm Plugins
                                   (npm/git)
```

## 五、设计哲学对比

```
Codex:
  "安全第一，扩展第二"
  → 规则引擎 + OS 沙箱做安全
  → Skills 和 MCP 做扩展
  → 但扩展能力有限——你不能改 Codex 的行为，只能加东西

Pi-Mono:
  "一切都可以被替换"
  → Extension API 是核心设计
  → 甚至内置工具都可以被扩展覆盖
  → beforeToolCall 可以拦截任何工具
  → 最大的灵活性，最少的内置假设

Claude Code:
  "多种扩展机制，分层使用"
  → Skills: 注入知识（最轻量）
  → Hooks: 拦截和审计（中等）
  → Plugins: 添加功能（最重）
  → MCP: 连接外部服务（标准协议）
  → 每种机制有明确的使用场景
```

## 六、关键差异总结

| 维度 | Codex | Pi-Mono | Claude Code |
|------|-------|---------|-------------|
| **扩展主力** | MCP + Skills | Extension API | Hooks + Skills + MCP |
| **灵活度** | 低（受沙箱限制）| 最高（可替换一切）| 高（多种机制）|
| **安全性** | 最高（OS 级）| 最低（信任扩展）| 中（钩子 + 权限）|
| **学习成本** | 低（配置文件）| 中（API 学习）| 中（多种机制）|
| **UI 扩展** | 无 | 全面（组件级）| 有限（React 内置）|
| **包管理** | 无 | Pi Packages | npm Plugins |

## 七、给你的思考题

1. **为什么 learn-claude-code 的 Skill 用"两层加载"而不是全部放进 system prompt？**
   - 提示：10 个技能 × 5000 token = 50000 token 的 system prompt
   - 两层加载：名字列表只需 100 token，按需加载单个技能 5000 token

2. **Pi-Mono 的 Extension API 为什么比 Claude Code 的 Hooks 更灵活？**
   - Hooks：只能拦截/审计，不能添加 UI 或修改行为
   - Extension API：可以注册工具、命令、快捷键、UI 组件、事件处理器

3. **如果你要设计一个扩展系统，你会选哪种模式？**
   - 纯配置（Codex）→ 简单但有限
   - 编程 API（Pi-Mono）→ 强大但需要学习
   - 混合（Claude Code）→ 灵活但复杂
